export const meta = {
  title: `Pages`
}

Pages live in `src/pages` and are either `.mdx` or `.jsx` files.

Create a new page with the `page` package script:

```bash
$ yarn page path/to/my-new-page
# ðŸ–¨ Created: src/pages/path/to/my-new-page.mdx
```

Every page is a React component that receives two props:

- `meta`: the page's own `meta` object
- `pages`: an object containing all pages, keyed by pathname

## JSX pages

You can author pages as a plain React component. The page component must be the `default` export.

```jsx
export const meta = {
  title: 'My page title'
}

export default function MyPage ({ meta, pages }) {
  return <div>Hello world</div>
}
```

## MDX pages

See the [MDX docs](https://mdxjs.com/) for the Markdown + JSX syntax you can use in `.mdx` files.

```mdx
export const meta = {
  title: 'My page title',
  description: 'Description for search engines',
  tags: ['post'],
  date: '2021-04-01'
}

# Hello world

Mix JSX into your Markdown.

<p>{props.meta.description}</p>

import Select from 'react-select'

<Select options={[
  { value: 'chocolate', label: 'Chocolate' },
  { value: 'strawberry', label: 'Strawberry' }
]} />

<strong>{props.page.description}</strong>
```

### Syntax highlighting

Tropical just follows MDX's [syntax highlighting guide](https://mdxjs.com/guides/syntax-highlighting). The provided `<CodeBlock>` component uses [react-syntax-highlighter](https://github.com/react-syntax-highlighter/react-syntax-highlighter).

## Page `meta`

The optionally exported `meta` object can contain any properties you wish, or none at all.

If present, the following properties will be used by Tropical (but you're free to use them too).

- `title` *(string)* â€” used for the page's `<title>` and the JSON Feed item's `title` (if the page appears in the feed)
- `description` *(string)* â€” used for the page's `<meta name='description'>`
- `tags` *(array of strings)* â€” the default JSON feed ([`src/feeds/posts.jsx`](https://github.com/bensmithett/tropical/blob/HEAD/src/feeds/posts.jsx)) only includes pages that include a `'post'` tag
- `Layout` *(component)* â€” alternative [layout component](/docs/layouts)
- `date` *(string [parsable](https://day.js.org/docs/en/parse/string) by [day.js](https://day.js.org/))* â€” default sorting mechanism for [`pagesWithTag`](https://github.com/bensmithett/tropical/blob/HEAD/src/utils.js) (see below)

### Working with `meta.tags`

You can use `meta.tags` for anything.

One obvious use is creating collections of pages â€” for example, a date-ordered collection that includes blog posts but not your homepage page. Tropical provides [`pagesWithTag`](https://github.com/bensmithett/tropical/blob/HEAD/src/utils.js) for this common scenario.

```jsx
import { pagesWithTag } from '../utils'

const blogPosts = pagesWithTag(props.pages, 'post')

<ul>
  {blogPosts.map(({ meta, urlPath }) => (
    <li>
      <a href={urlPath}>{meta.title}</a>
    </li>
  ))}
</ul>
```
